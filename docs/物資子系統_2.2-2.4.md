## 2.2 技術

**物資子系統：**

| 層級 | 技術 | 說明 |
| --- | --- | --- |
| 前端框架 | Flutter 3.x（Dart） | 跨平台 App、Material 3 高齡友善 UI |
| 前端狀態 | Riverpod | 非同步狀態、Realtime 訂閱 |
| 前端路由 | GoRouter、RoleGuard | 長輩／志工角色分流 |
| 後端平台 | Supabase BaaS | Auth、PostgreSQL、Realtime、Storage |
| 無伺服器函式 | Supabase Edge Functions（Deno + TypeScript） | NLU 補強、FCM 推播、閒聊 |
| 身分驗證 | Supabase Auth（Email + OTP） | 單一帳號；`profiles.role` 區分 elder／volunteer |
| 權限控管 | PostgreSQL RLS | 長輩僅存取自身資料；志工可協作訂單與履行 |
| 語音輸入 | speech_to_text | 中文聽寫；`pauseFor=3.5s` 適應高齡語速 |
| 語音輸出 | flutter_tts（zh-TW） | 小幫手回覆播報 |
| NLU 引擎 | Hybrid NLU（端側規則 + Edge 補強） | L1 關鍵字 → L2 正則 → L3 同義詞；低信心呼叫 `parse_shop_nlu` |
| 離線佇列 | Hive、connectivity_plus | 斷網暫存草稿；`clientRequestId` 冪等重傳 |
| 即時同步 | Supabase Realtime（WebSocket） | 訂閱 orders、demand_records 變更 |
| 推播 | firebase_messaging、FCM HTTP v1 | Android 鎖屏通知（Edge `send_shop_push`） |
| 生成式 AI | Edge `assistant_casual_chat`（Gemini） | 僅閒聊路徑；代購核心不依賴 LLM |
| 圖表 | fl_chart | 志工數據總覽訂單統計 |

## 2.3 後端模組設計

**物資子系統：**

| 模組／機制名稱 | 核心功能與設計說明 |
| --- | --- |
| Auth（身分驗證） | 支援 Email 登入／註冊；註冊時可勾選「社區志工」；系統依據使用者角色自動區分長輩與志工 |
| PostgreSQL（資料儲存） | 結構化儲存物資子系統核心資料表，包含代購訂單、需求草稿、標準商品主檔、通路參考價、對話紀錄、社區據點與推播憑證登記等 |
| Realtime（即時同步） | 透過即時連線建立資料串流；長輩送出需求或志工更新訂單進度時，雙方 App 畫面秒級同步，無需手動重新整理 |
| Edge Function（邊緣運算） | 負責三大雲端任務：語意補強（口語不清楚時再次解析）、推播通知（接單與送達提醒）、閒聊回覆（與代購路徑分開） |
| RLS（資料列權限控管） | 長輩僅能存取自己的需求草稿與訂單；志工僅可讀寫協作範圍內的訂單與品項履行資料；小幫手對話紀錄僅本人可見 |
| 本機離線暫存 | 結合手機本機儲存與連線偵測；斷網時先將需求存於手機，恢復連線後自動上傳草稿，避免重複建立相同品項 |
| 端側語意理解 | 手機端先行判斷「要買什麼、買幾個」；語句不完整時反問補充；一句話說多項商品可自動拆分 |
| 今日採買彙整 | 資料庫後端依據點與日期，即時將多位長輩需求合併成「今日採買清單」，取代志工紙筆抄寫 |
| 個人化推薦 | 串接購買與推薦紀錄，依長輩常買類別、平價偏好與歷史購買頻率，顯示三張推薦卡片協助快速選品 |

## 2.4 資料庫設計

**物資子系統：**

#### 代購訂單與需求

| 資料表 | 中文名稱 | 核心用途 |
| --- | --- | --- |
| orders | 代購訂單主檔 | 儲存長輩正式送出的代購單，含處理狀態、所屬據點、負責志工、送達時間與備註 |
| order_items | 訂單明細 | 記錄每一張訂單內的商品名稱、數量與參考單價 |
| demand_records | 需求草稿主檔 | 儲存長輩尚未送出或已轉正式單前的需求單，可對應到最終訂單 |
| demand_record_items | 需求明細 | 記錄各品項名稱、數量，以及志工是否已購買、已送達或改買替代品 |
| order_delivery_events | 配送進度紀錄 | 記錄訂單從建立、接單、採買到送達各階段時間點，供長輩查看進度 |
| demand_fulfillment_events | 品項履行紀錄 | 追蹤單一品項狀態變更歷程，便於志工與據點事後查核 |

#### 商品、價格與推薦

| 資料表 | 中文名稱 | 核心用途 |
| --- | --- | --- |
| product_categories | 商品類別 | 定義衛生紙、雞蛋等採買分類，供柑仔店選品與推薦使用 |
| product_brands | 品牌與規格 | 儲存各品牌名稱、包裝規格與參考價格 |
| product_items | 標準商品主檔 | 將類別、品牌、規格整合為系統可辨識的標準品項 |
| product_aliases | 口語同義詞 | 將長輩常說的別名（如「衛生紙」）對應到標準商品，協助語音理解 |
| price_references | 參考價格 | 儲存全聯等通路參考價，供小幫手查價與價格頁顯示 |
| purchase_events | 購買紀錄 | 記錄長輩過去購買行為，作為個人化推薦依據 |
| recommendation_logs | 推薦紀錄 | 記錄系統曾向長輩推薦哪些商品，供日後分析與優化 |

#### 智慧小幫手與據點協作

| 資料表 | 中文名稱 | 核心用途 |
| --- | --- | --- |
| assistant_chat_sessions | 對話紀錄主檔 | 儲存長輩與小幫手每一場對話的內容與時間 |
| chat_histories | 對話歷史檢視 | 供 App 讀取對話列表的檢視表，底層資料來自對話紀錄主檔 |
| clarification_sessions | 澄清對話狀態 | 當長輩語句不完整時，暫存反問與補充資訊直至需求釐清 |
| location_points | 社區據點 | 儲存明德里活動中心等據點名稱、地址與聯絡方式 |
| location_assets | 據點物資清單 | 記錄據點現有物資數量，供志工數據總覽檢視 |
| daily_shopping_lists | 今日採買清單 | 可選擇性儲存當日彙整後的採買清單（主流程亦支援即時彙整） |
| device_tokens | 推播裝置登記 | 記錄使用者手機推播識別，用於發送接單與送達通知 |
| push_notifications_log | 推播發送紀錄 | 記錄每次通知是否成功送達，便於除錯與追蹤 |

**權限設計摘要**：長輩僅能讀寫自己的需求單與訂單；志工可讀寫協作範圍內的訂單、品項履行與據點資料；對話紀錄僅本人可存取，避免隱私外洩。
